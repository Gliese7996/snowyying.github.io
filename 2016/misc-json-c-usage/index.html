json-c 使用总结

2016-09-27
Misc
1 json_object_get / json_object_put 函数

如果对json对象显式调用了json_object_get，之后必须成对调用json_object_put，否则将导致该json对象所占用内存泄漏。

2 json_object_new_XXX 系列函数

json_object_new_XXX系列函数用来创建XXX类型的json对象，创建的json对象默认引用计数为1，因此在该对象使用完后也需要调用一次json_object_put来把引用计数置0，从而释放内存。

3 json_tokener_parse 函数

struct json_object* json_tokener_parse(const char *str);
json_tokener_parse将符合json格式的字符串构造为一个json对象，构造的json对象默认引用计数为1，同样需要在使用完后对该对象调用一次json_object_put。

4 is_error宏

is_error(json)
如果传入的字符串是非法的json格式，错误判断应该使用is_error宏，而非 if(NULL != json)，CGI中目前有很多这种错误用法(虽然好像没引发什么问题)

5 json_object_object_XXX函数

  • void json_object_object_del(struct json_object* json, const char *key);
    从json对象中删除键值为key的子对象，并释放该子对象及键值所占的内存（注：可能有文档说json_object_object_del只是删除而不释放内存，但实际上这是错误的）。

  • struct json_object* json_object_object_get(struct json_object* json, const char *key); 从json中获取键值为key的子对象。错误判断同样应该用is_error(json)宏。

  • void json_object_object_add(struct json_object* json, const char *key, struct json_object *val); 更新键值为key的子项的值。整个过程实际上是这样的：先从json中删除并释放键值key及其值的内存，然后重新分配内存添加键值和新的值，所以json_object_object_add是包含json_object_object_del操作的。

6 json_object_get_XXX系列函数

这类函数只是获取json对象的实际数据内容，不更新引用计数，也不分配内存。
