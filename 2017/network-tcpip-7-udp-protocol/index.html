TCP/IP 协议 - UDP 协议

2017-01-09
Network

应用程序必须关心IP数据报的长度, 如果它超过网络的MTU, 则要对IP数据报进行分片.

UDP 头

端口号表示发送和接收进程(TCP/UDP的端口号是独立的)

  • IP 长度: 数据报文全长.
  • UDP 长度: UDP头 + UDP数据 的字节长度(>= 8). [UDP 长度是冗余的, = IP报文长度 - IP头]
UDP 校验和

UDP校验和是可选的; TCP校验和是必须的.

  • IP 校验和: 只覆盖IP头
  • UDP 校验和: 覆盖 UDP头 + UDP数据

方法: 16 bit 的二进制反码和

  1. 数据包长度存在奇数, 最后字节补0填充(仅用于校验)
  2. UDP 和 TCP 都存在12 Byte的伪头部(仅用于校验)
